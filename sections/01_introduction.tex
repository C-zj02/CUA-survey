\section{Introduction}
\label{sec:intro}

The rapid rise of \emph{agentic AI} has begun to shift large language models from passive responders into systems that can \emph{perceive}, \emph{reason}, and \emph{act} to complete complex tasks in interactive environments. This shift is enabled not only by stronger language modeling, but also by multimodal perception, tool and code execution interfaces, and increasingly realistic evaluation environments. A prominent instantiation of this trend is \textbf{Computer-Use Agents (CUAs)}: agents that close the loop between observing a computer environment (e.g., web pages, desktop operating systems, mobile apps) and taking environment-grounded actions. In practice, CUAs often operate through a \emph{hybrid action interface}---combining GUI interactions (clicking, typing, scrolling) with programmatic operations such as API calls or command execution---to accomplish end-to-end user goals.

Compared with dialogue-only assistants, CUAs substantially expand the real-world impact surface of AI systems. Model errors or misalignment no longer remain confined to incorrect text, but can translate into concrete side effects such as unintended purchases, data deletion, permission changes, or unsafe command execution. Accordingly, recent work has moved from controlled and reproducible web benchmarks (Mind2Web, WebArena) to more open-ended, real-computer settings (OSWorld), where agents must handle unfamiliar applications, non-stationary interfaces, and long-horizon cross-application procedures~\cite{deng2023mind2web,zhou2023webarena,xie2024osworld}.

Despite rapid capability gains, the CUA ecosystem remains fragmented across architectural design, evaluation environments, and safety analysis, motivating a more systematic understanding of this emerging landscape.

\textbf{First, architectural heterogeneity} makes it difficult to compare and reason about existing approaches. Some agents rely on DOM or accessibility trees for structured state representations~\cite{deng2023mind2web,zhou2023webarena}, others operate purely from pixels~\cite{lu2024omniparser, hong2024cogagent}, while hybrid systems combine visual grounding with structural priors~\cite{baechler2024screenai}. Decision modules further vary from single-step action prediction to memory-augmented reasoning, reflection loops, and multi-agent planning~\cite{xie2024osworld,zhang2025ufo}. These choices introduce non-obvious trade-offs in portability, robustness, latency, and error recovery, and they interact differently with deployment constraints and runtime failures.

\textbf{Second, CUAs introduce qualitatively new safety and governance challenges} compared with dialogue-only LLM systems. Because CUAs can directly manipulate interfaces and invoke high-impact operations, they expose new attack surfaces and failure modes, including indirect prompt injection through untrusted content, interface deception that misleads perception, and privilege or tool misuse during multi-step task execution. Recent safety benchmarks indicate that misuse, prompt injection, and unsafe action selection remain open problems for GUI-capable agents~\cite{kuntz2025harm,tur2025safearena,liu2025wainjectbench}. As CUAs become more autonomous and operate over longer horizons, traditional static safety assumptions and governance practices are increasingly strained, calling for lifecycle-aware analysis and system-level controls~\cite{lazer2026survey,adapala2025aegis}.

To address these challenges, we present a \textbf{lifecycle-oriented synthesis of the CUA ecosystem} that connects architectural design, operational risks, and governance mechanisms. Specifically, we make three contributions:
\begin{itemize}
  \item \textbf{Integrated architecture--lifecycle analysis.} We analyze the design space of computer-use agents through a perception--decision--execution pipeline and connect it with a four-stage lifecycle spanning \emph{Creation, Deployment, Execution, and Maintenance}. This integrated perspective clarifies how architectural choices shape capabilities, robustness, and failure modes across system stages.
  \item \textbf{Security threat modeling for computer-use agents.} We \emph{introduce} a lifecycle-oriented threat modeling perspective for CUA systems, characterizing emerging attack surfaces such as prompt injection, interface deception, tool misuse, and memory poisoning, and analyzing how these risks arise from both model behavior and adversarial environments.
  \item \textbf{Governance mechanisms for the CUA ecosystem.} We discuss governance strategies for deploying computer-use agents in open computing environments, including benchmarking and evaluation frameworks, auditing and provenance mechanisms, permission and sandboxing controls, and accountability across stakeholders.
\end{itemize}

Taken together, our goal is not only to summarize representative systems and benchmarks, but to provide an integrative map that connects \emph{how} CUAs are built, \emph{where} they fail in practice, and \emph{what} governance mechanisms are required as agents operate at scale in open ecosystems.
