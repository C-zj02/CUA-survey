\section{A Comprehensive Lifecycle Taxonomy and Architectural Analysis of Computer-Use Agents}
\label{sec:taxonomy}
\label{sec:lifecycle}

We organize representative CUA studies into a four-stage lifecycle taxonomy: \textit{Creation}, \textit{Deployment}, \textit{Execution}, and \textit{Maintenance}. This structure links engineering choices to where capability bottlenecks and security failures originate, and supports stage-wise analysis of architecture, integration, runtime, and update trade-offs.

\subsection{Creation Stage: Grounding Semantics in Multimodal Infrastructure}
The Creation stage fundamentally addresses the "semantic-to-spatial" alignment problem: training foundation models to inherently understand Graphical User Interfaces (GUIs). This stage marks a shift from heuristic OCR pipelines to native Multimodal Large Language Models (MLLMs) and scalable sandboxes.

\paragraph{Multimodal Foundation Architectures}
Early web-agent pipelines reported context-window pressure when processing long DOM trees, motivating filtering and compression strategies~\cite{deng2023mind2web}. Recent architectures shift towards native pixel-level processing. CogAgent \cite{hong2024cogagent} addresses miniature UI feature extraction via a high-resolution cross-attention module. Fuyu-8B \cite{fuyu-8b} pioneers a highly efficient decoder-only architecture, linearly projecting image patches directly into the LLM's embedding space to bypass traditional image encoders. To embed deep layout comprehension, Pix2Struct \cite{hwang2023pix2struct} introduces a screenshot parsing pre-training objective, while ScreenAI \cite{baechler2024screenai} maps pixels to structured UI layouts. Furthermore, SeeClick \cite{cheng2024seeclick} focuses exclusively on spatial regression, pre-training models to accurately output $(x,y)$ coordinates through pure visual point-and-click data. Across these designs, a common trade-off appears: structure-heavy pipelines improve deterministic grounding, while pixel-native models improve portability but depend more on robust visual localization.

\paragraph{Data Synthesis and Environmental Sandboxes}
Interaction data dictates the upper bound of CUA capabilities. To tackle web data noise, Mind2Web \cite{deng2023mind2web} implements a DOM-filtering pipeline distilling bloated HTML into actionable elements, while AITW \cite{rawles2023android} scales mobile data by curating millions of real-world Android screen-action pairs. To optimize the mixture of generic LLM data and agent-specific trajectories, AgentTuning \cite{zeng2024agenttuning} proposes an instruction-tuning methodology that prevents catastrophic forgetting of foundational reasoning. Because static datasets fail to capture dynamic UI state changes, researchers have built self-hosted sandboxes. WebArena \cite{zhou2024webarena} provides isolated, highly realistic web environments (e.g., e-commerce, CMS). Taking this to the system level, OSWorld \cite{xie2024osworld} architects a full-stack Ubuntu sandbox, migrating training from static sequence prediction to interactive environmental grounding.
These efforts reveal another recurring trade-off: large static corpora improve data breadth, while interactive sandboxes improve fidelity to non-stationary runtime behavior at higher engineering cost.

\subsection{Deployment Stage: Cross-Platform Abstraction and Action Mapping}
\label{sec:deployment}
Deployment investigates the architectural mechanisms to mount a CUA onto arbitrary operating systems, balancing \textit{intrusive API-dependent frameworks} against \textit{pure-vision architectures}.



\paragraph{System-Agnostic Migration Frameworks}
Modern deployment advocates for universal applicability over platform-specific backdoors (e.g., Selenium). The Cradle framework \cite{wei2024cradle} abstracts all interactions into a universal interface—screen pixels as input, generic mouse/keyboard events as output—enabling zero-shot deployment in complex environments like 3D gaming. Similarly, Mobile-Agent \cite{wang2024mobile} and AppAgent \cite{zhang2024appagent} achieve accessibility-tree-free deployment on smartphones via OCR and ADB simulation, introducing an "exploration stage" to map unfamiliar apps. At the desktop level, OS-Copilot \cite{wu2024oscopilot} integrates bash execution, Python interpreters, and GUI interactions for Linux/macOS, while ScreenAgent \cite{niu2024screenagent} utilizes VNC protocols to continuously stream and control remote desktop pixel arrays. To manage concurrency, AIOS \cite{mei2024aios} conceptualizes an LLM agent operating system kernel, handling resource allocation and context isolation across multiple deployed agents.

\paragraph{Action Space Abstraction and Middleware}
Translating high-dimensional LLM intent into precise coordinate clicks is error-prone. WebVoyager \cite{he2024webvoyager} elegantly circumvents this spatial regression weakness via a Set-of-Mark (SoM) visual prompting strategy, overlaying numeric bounding boxes directly on screenshots. OmniACT \cite{kapoor2024omniact} deploys an intermediate compilation layer that translates semantic goals into executable PyAutoGUI scripts. Microsoft's UFO \cite{zhang2024ufo} utilizes a dual-agent architecture (application-selector and action-executor) to seamlessly navigate Windows OS, while Ferret-UI \cite{you2025ferretui} provides an any-resolution adaptive deployment mechanism, ensuring visual grounding remains robust regardless of the specific device's aspect ratio.

\subsection{Execution Stage: Long-Horizon Reasoning and Fault Tolerance}
\label{sec:execution}
During online execution, CUAs face the "cascading failure" problem. Execution architectures heavily index on dynamic planning and state tracking to prevent localized errors from invalidating entire task trajectories.

\paragraph{Hierarchical Planning and Search}
The ReAct paradigm \cite{yao2023react} successfully interleaves reasoning and acting, establishing step-by-step interpretability. To overcome ReAct's Markovian execution bias in complex tasks, Tree of Thoughts (ToT) \cite{yao2023tree} introduces tree-search architectures (BFS/DFS), allowing agents to backtrack from dead-end UI states. LUMOS \cite{yin2024lumos} decouples macro-planning from micro-execution, preventing low-level visual noise from polluting the reasoning context. AutoUI \cite{zhan2024autoui} utilizes action-sequence prediction to anticipate multi-step workflows, PolySkill \cite{yu2025polyskill} introduces polymorphic abstractions to decouple intent from specific UI implementations, and LlamaTouch \cite{e2024llamatouch} optimizes on-device inference to reduce execution latency on mobile interfaces.

\paragraph{Visual State Tracking and Self-Correction}
Executing actions blindly without state verification causes catastrophic hallucination. Reflexion \cite{shinn2023reflexion} injects a linguistic episodic memory buffer, forcing agents to generate verbal reinforcement logs after errors. Synapse \cite{zheng2024synapse} uses trajectory-as-exemplar prompting, dynamically retrieving successful historical executions to guide the next step. WebAgent \cite{gur2023webagent} utilizes a multi-actor heuristic to parse dynamically changing DOM snippets during runtime, while ColorAgent \cite{li2025coloragent} focuses on long-horizon dynamic correction, recognizing environmental anomalies and actively recalculating its path.

\subsection{Maintenance Stage: Combating UI Decay and Context Overflow}
\label{sec:maintenance}
The final stage addresses vulnerabilities over time. As software interfaces undergo natural decay (e.g., UI redesigns), CUAs require mechanisms for continuous evaluation, security auditing, and lifelong learning.



\paragraph{Dynamic Benchmarking and Security}
Static datasets become obsolete rapidly. VisualWebArena \cite{koh2024visualwebarena} pushes evaluation frontiers by incorporating complex visual dependencies (e.g., interpreting charts). AgentBench \cite{liu2024agentbench} provides a multi-dimensional radar for OS capabilities. To tailor reinforcement learning to GUI tasks, CUARewardBench \cite{lin2025cuarewardbench} calibrates reward models specifically for computer-use trajectories. Moving towards domain-specific and industrial maintenance, EconWebArena \cite{liu2025econwebarena} evaluates agents on live financial/economic platforms, while WorkArena \cite{drouin2024workarena} assesses robustness in enterprise SaaS environments (e.g., ServiceNow). To counter static benchmark decay, WebCanvas \cite{pan2024webcanvas} extends the Mind2Web paradigm to mutating live web environments and introduces online regression testing via node replay. Furthermore, SafeArena \cite{kumar2025safearena} establishes critical security benchmarks, auditing resistance against prompt injections and destructive OS commands.

\paragraph{Episodic Memory and Continual Learning}
Continuous execution leads to context saturation. MemGPT \cite{packer2024memgpt} provides a foundational fix by borrowing memory paging from traditional OS, allowing agents to page context in and out of a finite context window to prevent catastrophic forgetting. The same WebCanvas framework \cite{pan2024webcanvas} utilizes runtime node-replay mechanisms to automatically log dynamic web changes, generating fresh training data organically. Finally, ExpeL \cite{zhao2023expel} facilitates experiential learning, allowing the agent to continuously fine-tune its internal policies via self-play, ensuring continuous evolution without computationally prohibitive retraining.
