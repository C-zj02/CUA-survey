\section{Architectural Evolution of Computer-Use Agents: A Tri-Layer Taxonomy}
\label{sec:architecture}

The rapid advancement of Multimodal Large Language Models (MLLMs) has catalyzed the development of Computer-Use Agents (CUAs), transitioning them from narrow, domain-specific scripts to generalist systems capable of autonomous Graphical User Interface (GUI) navigation. To systematically evaluate the current state-of-the-art, we deconstruct the CUA architecture into three foundational layers: the \textit{Perception Layer}, the \textit{Decision Layer}, and the \textit{Execution Layer}. This taxonomy elucidates the data flow from raw pixel ingestion to physical or virtual action instantiation.

As illustrated in Fig.~\ref{fig:tri_layer_architecture}, the three layers form a closed control loop: environment observations are grounded into structured state representations, transformed into executable plans, and then fed back through updated system states after action execution.

\subsection{Perception Layer: Structured Reconstruction of Screen States}
The primary objective of the Perception Layer is to bridge the semantic gap between human-readable visual layouts and machine-processable structured representations (i.e., GUI Grounding). Based on the source of state observation, existing methodologies can be categorized into three dominant paradigms.

\textbf{DOM/A11y-based Parsing.} This paradigm relies on the explicit extraction of the underlying Document Object Model (DOM) or Accessibility (A11y) trees provided by the operating system or browser environment. Representative works such as Mind2Web \cite{deng2023mind2web} introduce candidate element filtering mechanisms to truncate excessively long DOM trees into manageable context windows. Similarly, WebArena \cite{zhou2024webarena} heavily leverages structured HTML representations to achieve deterministic state capture. While highly accurate in retrieving text and metadata, this approach suffers from severe cross-platform fragility and fails entirely in rendering-heavy applications like commercial video games or virtual machines.

\textbf{Pure Vision-based Parsing.} To circumvent the limitations of system APIs, a growing body of research treats GUI interaction purely as a computer vision task. OmniParser \cite{omniparser} eliminates DOM dependency entirely, parsing raw screenshots into structured bounding boxes and text descriptions using pure visual cues. CogAgent \cite{hong2024cogagent} employs a dual-branch cross-attention mechanism to process high-resolution (up to 1120x1120) screen inputs, capturing minute UI icons effectively. Fuyu-8B \cite{bavishi2023fuyu} flattens image patches directly into transformer layers to enhance resolution flexibility, while Qwen-VL \cite{qwenvl} and UI-ViE \cite{uivie} have proven highly effective in zero-shot bounding box localization and complex visual information extraction (e.g., tables and lists).

\textbf{Hybrid Modality Grounding.} State-of-the-art models increasingly fuse visual and structural priors. ScreenAI \cite{baechler2024screenai} integrates optical character recognition (OCR) with screen layout comprehension to generate rich-text UI representations. Ferret-UI \cite{you2025ferretui} embeds spatial coordinate predictions directly into multimodal token streams, enhancing mobile UI grounding. Recent efforts like SeeClick \cite{cheng2024seeclick}, TRISHUL \cite{trishul}, and the ScreenParse benchmark \cite{screenparse} advocate for holistic hierarchical screen parsing (HSP), transitioning perception from sparse point-clicking to dense, full-screen topological reconstruction.

\subsection{Decision Layer: Intent Reasoning and Long-Horizon Planning}
Acting as the cognitive core, the Decision Layer decomposes high-level natural language instructions into sequential operational trajectories. As task horizons extend, simple step-by-step mapping has evolved into sophisticated cognitive architectures.
Long-horizon GUI tasks accumulate execution errors and context degradation, motivating both (i) correction loops and (ii) hierarchical decomposition in modern decision modules~\cite{xie2024osworld,LPSBench2026}.

\textbf{Memory-Augmented Planning.} To navigate novel environments without exhaustive trial-and-error, agents increasingly rely on episodic memory. AppAgent \cite{zhang2024appagent} explicitly separates an ``exploration stage'' from an ``execution stage,'' allowing the agent to compile a vectorized memory bank of UI interactions that are retrieved during complex task planning. InfiGUIAgent \cite{infiguiagent} similarly utilizes native reasoning and reflection, recalling past successful trajectories to avoid entering infinite operational loops. Generalist visual agents like OSWorld baselines \cite{xie2024osworld} heavily rely on context-aware retrieval to maintain state consistency across prolonged interactions.

\textbf{Closed-loop Reflection and Correction.} Cumulative execution errors are fatal in multi-step CUA tasks. Consequently, robust decision layers incorporate ``Execution-Evaluation-Correction'' loops. WebVoyager \cite{he2024webvoyager} utilizes a hybrid visual-textual trajectory reasoning mechanism, enabling the agent to backtrack upon detecting a misclick. Auto-GUI \cite{autogui} enforces a ``Chain-of-Action'' prompting technique, compelling the model to articulate its deductive reasoning before outputting specific actions, thereby enhancing interpretability and self-correction. ReInAgent \cite{reinagent} extends this by integrating human-in-the-loop (HITL) active questioning to resolve ambiguous intents. Mobile-Agent \cite{wang2024mobile} further employs dynamic re-planning based on real-time visual feedback after every step.

\textbf{Hierarchical and Multi-Agent Decomposition.} For cross-application workflows, monolithic decision architectures often suffer from context degradation. UFO \cite{zhang2024ufo} mitigates this via a dual-agent framework, utilizing an Application Agent for macro-level software routing and an Action Agent for micro-level operations. CoAct-1 \cite{coact1} introduces a central orchestrator to decompose complex cross-application tasks into manageable sub-routines, representing a shift toward multi-agent GUI orchestration.

\subsection{Execution Layer: Instantiated Mapping of Action Spaces}
The Execution Layer translates the logical intent formulated by the Decision Layer into physical or API-driven manifestations. The design of the Action Space is critical for determining the agent's degree of freedom.

\textbf{Atomic Mouse/Keyboard Emulation.} The most generalized execution paradigm defines the action space strictly as low-level $(x, y)$ coordinate clicks, drags, and keystrokes. Cradle \cite{wei2024cradle} successfully demonstrated that agents restricted to pure atomic inputs could master highly complex environments like 3D gaming (e.g., Red Dead Redemption 2). DeepMind's AndroidEnv \cite{androidenv} also conceptualizes mobile interactions fundamentally as parameterized touch and swipe gestures, a standard adopted by pure visual agents \cite{uitars, guibee}. 
Security implication: this high-entropy action space maximizes flexibility but also enlarges the surface for misclicks, clickjacking, and action rebinding~\cite{ActionRebinding2026,Chameleon2025}.

\textbf{Code-as-Action and API Invocation.} Conversely, for software interaction, simulating clicks is often suboptimal compared to direct programmatic control. OSCopilot \cite{wu2024oscopilot} utilizes code generation (Python/Bash) to manipulate operating systems, demonstrating substantial efficiency gains in file management. The GUI-360 framework \cite{gui360} advocates for a hybrid space where agents can trigger built-in application macros (e.g., Excel VBA) alongside UI clicks to accelerate execution. WebArena \cite{zhou2024webarena} limits execution to standardized JavaScript browser events to ensure deterministic evaluation.
Security implication: higher-level execution compresses many actions into one call, so least privilege, provenance, and sandbox boundaries become more critical~\cite{Jones2025CUA,MCPSecurityBench2025}.

\textbf{Macro-Action and Continuous Control.} Bridging the gap between atomic inputs and API calls, recent architectures aim to abstract repetitive low-level behaviors. AppAgentX \cite{appagentx} introduces an evolutionary mechanism where the agent autonomously encapsulates frequently used sequences (e.g., \textit{Move $\rightarrow$ Click $\rightarrow$ Type}) into high-level macro-actions. Furthermore, ScreenAgent \cite{niu2024screenagent} and OmegaUse \cite{omegause} have formalized parameterized continuous control variables, allowing agents to execute complex, multi-stage scrolling or drag-and-drop operations with a single cohesive command.
Security implication: macro-abstractions reduce low-level action entropy but introduce verification and audit challenges when one command encapsulates multiple high-impact steps~\cite{AgentSecurityBench2025,VeriGuard2025}.
