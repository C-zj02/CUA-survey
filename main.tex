\begin{abstract}
This survey examines computer-use agents (CUAs) as they move from benchmarked web and OS assistants to agents that operate across browsers, desktops, mobile apps, and long-lived tool ecosystems. In these settings, capability and risk are determined not by model quality alone, but by the coupling among perception, decision-making, execution interfaces, memory, permissions, and deployment context. Existing work addresses these factors from separate perspectives, including GUI-agent architecture, benchmark design, and agent security, but lacks a synthesis of how they interact in deployed systems. We provide such a synthesis through a joint architecture--lifecycle lens. Specifically, we organize representative systems, benchmarks, and security studies with a tri-layer architecture of \emph{Perception}, \emph{Decision}, and \emph{Execution}, then connect that backbone to a four-stage lifecycle of \emph{Creation}, \emph{Deployment}, \emph{Operation}, and \emph{Maintenance}. This framework clarifies where capability bottlenecks are formed, where failures first surface, and where controls can be attached. Using it, we map major threats--including prompt injection, interface deception, over-privilege, memory poisoning, and supply-chain compromise--to lifecycle failure points and derive governance requirements centered on constrained action interfaces, provenance, sandboxing, runtime oversight, and continuous evaluation. CUAs should therefore be understood as lifecycle-dependent interactive systems whose reliability depends jointly on architectural design and operational control.
\end{abstract}
