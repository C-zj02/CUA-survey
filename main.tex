\begin{abstract}
Computer-use agents (CUAs) are evolving from benchmarked interface-following systems into deployable agents for real-world web, desktop, and mobile environments. This transition introduces a systems-level challenge: agent behavior is shaped not only by model capability, but also by the interaction among perception, planning, execution authority, memory, tool interfaces, and deployment constraints. Although existing studies have examined these factors through architectural analysis, benchmarking, and safety evaluation, a unified framework is still lacking to explain how they jointly determine capability, failure modes, and control requirements. To address this gap, we develop a two-dimensional analytical framework that combines a tri-layer architecture—Perception, Decision, and Execution—with a four-stage lifecycle consisting of Creation, Deployment, Operation, and Maintenance. This framework enables systematic analysis of how design choices in state representation, action abstraction, memory, and tool integration affect robustness and controllability across deployment stages. Based on this framework, we map major CUA security threats to concrete lifecycle-specific failure points and derive governance requirements centered on evaluation, provenance, sandboxing, auditing, and accountability. By treating CUAs as lifecycle-dependent interactive systems, this paper provides a unified foundation for understanding and governing computer-use agents in real-world environments.
\end{abstract}
